# Процесс сборки и подготовки к развертыванию

В данном разделе описаны шаги для сборки проектов, создания исполняемых файлов и подготовки архивов для установки или обновления системы BurstroyMonitoring.

## Основные этапы

### 1. Очистка предыдущих сборок
Перед началом новой сборки рекомендуется очистить папки с результатами предыдущих запусков.
*   **Скрипт:** `clean_release.ps1`
*   **Действие:** Удаляет содержимое папок `release` во всех проектах.

### 2. Сборка и публикация проектов
Для каждой целевой платформы предусмотрен отдельный PowerShell скрипт. Эти скрипты компилируют код и упаковывают каждый проект в индивидуальный ZIP-архив.

*   **Скрипты:**
    *   `publish_linux_arm64.ps1` — для систем на базе ARM64 (например, Raspberry Pi).
    *   `publish_linux_x64.ps1` — для стандартных Linux серверов.
    *   `publish_win_x64.ps1` — для Windows серверов.

**Что происходит внутри скрипта для каждого проекта (TCM, Worker, VideoMonitoring, Api):**
1.  **`dotnet publish`**: Компиляция проекта в режиме `Release`.
    *   Включается `PublishSingleFile=true` для сборки всего приложения в один исполняемый файл.
2.  **Создание ZIP-архива**: Скрипт автоматически собирает необходимые файлы в архив:
    *   Исполняемый файл (бинарник).
    *   Конфигурационный файл `appsettings.json`.
    *   Статические файлы из папки `wwwroot` (если применимо).
    *   Архив сохраняется в папку `release/[platform]/[ProjectName].zip`.

### 3. Создание общего бандла для развертывания
После того как все проекты собраны в индивидуальные архивы, создается единый установочный пакет (bundle), который содержит все компоненты системы.

*   **Скрипты:**
    *   `create_deploy_bundle_arm.ps1` — сборка общего архива для ARM64.
    *   `create_deploy_bundle_64.ps1` — сборка общего архива для x64.

**Результат:** В корневой папке `release` появится итоговый архив (например, `deploy_bundle_linux-x64.zip`), готовый к отправке на сервер.

## Краткая инструкция для обновления
1. Откройте терминал PowerShell в папке `admin/publish_cmd/`.
2. Выполните `.\clean_release.ps1`.
3. Выполните скрипт публикации для нужной платформы, например: `.\publish_linux_x64.ps1`.
4. Создайте итоговый бандл: `.\create_deploy_bundle_64.ps1`.
5. Полученный архив из папки `release` используйте для обновления системы на сервере.
