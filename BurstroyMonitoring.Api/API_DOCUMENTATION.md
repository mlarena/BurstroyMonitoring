# Документация BurstroyMonitoring Web API (v1)

Данный API предоставляет доступ к данным мониторинга , метеоданным и архивным показателям датчиков.

## Общая информация
- **Базовый URL:** `http://<host>:<port>/api/v1`
- **Формат данных:** JSON
- **Авторизация:** JWT Bearer Token (передается в заголовке `Authorization: Bearer {token}`)

---

## 1. Посты мониторинга (Monitoring Posts)

### Получить список всех постов
`GET /monitoring-posts`
Возвращает список всех объектов мониторинга.

### Получить данные конкретного поста
`GET /monitoring-posts/{id}`

### Получить список датчиков на посту
`GET /monitoring-posts/{id}/sensors`

### Получить текущую метеообстановку (Стандарт)
`GET /monitoring-posts/{id}/meteo-standart`
**Описание:** Возвращает агрегированные данные со всех метеодатчиков поста в унифицированном формате.

---

## 2. Архивные данные датчиков (Data)

Этот раздел API позволяет извлекать исторические показатели датчиков. Вы можете выбирать глубину архива и степень детализации данных (от сырых записей до усредненных значений за час).

### Как строится запрос
Все запросы к данным имеют общую структуру:
`GET /api/v1/data/sensors/{sensorId}/{type}/{interval}?days={count}`

1.  **`{sensorId}`** (обязательно) — уникальный числовой идентификатор датчика в системе. Его можно узнать через запрос `GET /monitoring-posts/{id}/sensors`.
2.  **`{type}`** (обязательно) — тип данных, которые вы хотите получить. Доступные типы:
    *   `dov` — Видимость (метеорологическая дальность видимости).
    *   `dspd` — Состояние дороги (температура покрытия, слой воды/льда/снега, сцепление).
    *   `dust` — Экология (концентрация пыли PM1, PM2.5, PM10).
    *   `iws` — Погода (температура воздуха, влажность, давление, ветер, осадки).
    *   `mueks` — Техническое состояние (напряжение АКБ, температура в шкафу, датчик открытия двери).
3.  **`{interval}`** (опционально) — уровень агрегации данных:
    *   *(пусто)* — возвращает **все** доступные записи без изменений.
    *   `hourly` — группирует данные по часам и возвращает среднее значение за каждый час.
    *   `ten-minute` — группирует данные по 10-минутным интервалам и возвращает среднее значение.
4.  **`days={count}`** (параметр запроса) — за сколько последних дней нужно вернуть данные.
    *   Если не указать, вернет данные за **1 день**.
    *   Пример: `?days=7` вернет архив за последнюю неделю.

---

### Подробное описание уровней детализации

#### А. Полные (сырые) данные
**URL:** `GET /data/sensors/{sensorId}/{type}`
Используйте этот метод, если вам нужны максимально точные данные для детального анализа или построения графиков за короткий промежуток времени (например, за несколько часов).
*   **Особенности:** Может возвращать очень большой объем данных, если указать много дней.
*   **Пример:** `GET /api/v1/data/sensors/10/iws?days=1` — все метеоизмерения датчика №10 за последние 24 часа.

#### Б. Почасовые интервалы (усредненные)
**URL:** `GET /data/sensors/{sensorId}/{type}/hourly`
Идеально подходит для аналитических отчетов и графиков за длительные периоды (неделя, месяц).
*   **Логика:** Система берет все измерения внутри каждого часа (например, с 10:00 до 11:00) и вычисляет их среднее арифметическое.
*   **Преимущество:** Быстрая загрузка и отсутствие лишнего "шума" на графиках.
*   **Пример:** `GET /api/v1/data/sensors/10/dust/hourly?days=30` — средняя концентрация пыли по часам за последний месяц.

#### В. 10-минутные интервалы (усредненные)
**URL:** `GET /data/sensors/{sensorId}/{type}/ten-minute`
"Золотая середина" между точностью и объемом данных.
*   **Логика:** Данные усредняются за каждые 10 минут (00-10, 10-20 и т.д.).
*   **Пример:** `GET /api/v1/data/sensors/10/dspd/ten-minute?days=3` — динамика состояния дороги за 3 дня с шагом в 10 минут.

## 3. Примеры использования

### Запрос данных через curl
```bash
curl -X GET "http://localhost:5000/api/v1/data/sensors/15/iws/hourly?days=7" \
     -H "Authorization: Bearer YOUR_TOKEN_HERE"
```

### Пример ответа (IWS Hourly)
```json
{
  "sensorId": 15,
  "serialNumber": "IWS-12345",
  "postName": "Пост №1 (Север)",
  "measurements": [
    {
      "receivedAt": "2024-05-07T10:00:00Z",
      "environmentTemperature": 15.5,
      "humidityPercentage": 62.1,
      "windSpeed": 3.4
    },
    {
      "receivedAt": "2024-05-07T11:00:00Z",
      "environmentTemperature": 16.2,
      "humidityPercentage": 60.5,
      "windSpeed": 4.1
    }
  ]
}
```

---

## 4. Коды ответов
- `200 OK` — Запрос выполнен успешно.
- `401 Unauthorized` — Отсутствует или невалидный токен.
- `404 Not Found` — Датчик или пост не найден.
- `500 Internal Server Error` — Ошибка на стороне сервера или БД.
