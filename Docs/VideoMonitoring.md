# Модуль видеонаблюдения и фиксации событий

Модуль предназначен для мониторинга объектов через RTSP-видеокамеры, автоматического создания архива снимков и управления оборудованием.

## Архитектура модуля

Модуль состоит из трех основных компонентов:
1. **Веб-интерфейс (TCM)**: Управление камерами, просмотр прямого эфира и работа с архивом.
2. **Фоновый воркер (VideoMonitoring)**: Автоматический захват кадров по расписанию.
3. **База данных**: Хранение настроек камер и метаданных снимков.

---

## 1. Фоновый воркер (Worker.cs)
Работает как независимая служба (BackgroundService).
- **Интервал проверки**: настраиваемая пользоватлей (по умолчанию каждые 600 секунд).
- **Логика**: 
    - Проверяет параметр интервал опроса для каждой камеры.
    - Если время пришло, запускает сервис для захвата одного кадра из RTSP-потока.
    - Сохраняет файл в `wwwroot/snapshots/{CameraId}/`.
   
---

## 2. Веб-интерфейс (CamerasController)

### Главная страница (Index)
- Отображает список всех подключенных камер в виде  карточек.
- Показывает превью последнего снимка.
- **Статистика**: Выводит общее количество камер и суммарное количество снимков, сделанных за текущие сутки, с детализацией по каждой камере.

### Просмотр камеры (Details)
- **Прямой эфир**: Трансляция потока в реальном времени. При загрузке отображается последний скриншот, который мгновенно перекрывается живым видео после установки соединения.
- **Ручной снимок**: Кнопка «Сделать снимок» позволяет мгновенно зафиксировать кадр вне очереди воркера.
- **История**: Отображает 6 последних снимков камеры в виде компактной сетки.

### Архив снимков (Snapshots)
- Позволяет фильтровать снимки по камере и диапазону дат.
- **Автообновление**: Режим автоматической перезагрузки страницы каждые 30 секунд для мониторинга новых поступлений в реальном времени.
- **Выгрузка**: Кнопка формирования ZIP-архива. Файлы в архиве именуются по шаблону `ГГГГММДД_ЧЧММСС_имя.jpg` для удобства сортировки.

---

## 3. Технические особенности
- **Хранение**: Изображения хранятся в файловой системе, в БД хранятся только относительные пути.
- **Безопасность**: Поддержка авторизации в RTSP-потоках (автоматическая сборка URL с логином/паролем).
- **Зависимости**: Для работы захвата кадров на сервере должен быть установлен `ffmpeg`.
