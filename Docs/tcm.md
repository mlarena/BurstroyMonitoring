# Руководство пользователя Системы Мониторинга BurstroyMonitoring.TCM

## 1. Введение
Система предназначена для сбора, хранения, визуализации и анализа данных с метеорологических датчиков и оборудования (IWS, DSPD, DOV, DUST, MUEKS). Также система обеспечивает управление справочной информацией, администрирование пользователей и аудит действий.

---

## 2. Справочники (Настройка системы)
*Путь: Панель навигации -> Справочники.*

В данном разделе настраивается структура системы перед началом сбора данных.

### 2.1. Посты мониторинга (`MonitoringPosts`)
**Назначение:** Создание и редактирование физических или логических точек установки оборудования (например, "Мост через р. Волга", "Северный пост").
**Возможности:**
- **Создать новый:** Указать название, адрес, описание.
- **Координаты:** Ввести широту и долготу для отображения на карте.
- **Тип:** Отметить "Мобильный пост", если оборудование перемещается.
- **Статус:** Активировать или деактивировать пост.

### 2.2. Типы датчиков (`SensorTypes`)
**Назначение:** Справочник типов оборудования.
**Важно:** Заполняется автоматически при старте системы. Доступен только для просмотра. Включает типы:
- **DSPD** — Состояние дорожного полотна.
- **IWS** — Параметры атмосферы.
- **DOV** — Оптическая видимость.
- **DUST** — Концентрация пыли.
- **MUEKS** — Управление энергоснабжением и контроль сети.

### 2.3. Датчики (`Sensors`)
**Назначение:** Регистрация конкретных единиц оборудования в системе.
**Возможности:**
- **CRUD операции:** Добавление, редактирование, удаление датчика.
- **Параметры подключения:**
    - **Серийный номер (S/N):** Уникальный идентификатор устройства.
    - **Тип датчика:** Привязка к одному из типов (IWS, DSPD и т.д.).
    - **Пост мониторинга:** Привязка к посту.
    - **URL API:** Эндпоинт, откуда система-сборщик берет данные.
    - **Интервал опроса:** Периодичность сбора данных (в секундах).

## 3. Результаты сбора данных (Просмотр)
*Путь: Панель навигации -> Данные.*

Все разделы данных работают в режиме "Только чтение". Здесь отображаются данные, полученные от датчиков.

### 3.1. Таблицы данных (DovData, IwsData, DspdData, DustData, MueksData)
**Назначение:** Просмотр сырых показаний датчиков.
**Функции интерфейса:**
- **Таблица:** Отображает основные параметры (Время, S/N, Название, Основные показатели).
- **Детали (📊):** Просмотр всех параметров конкретной записи (температура, влажность, сцепление, статус лазера и т.д.).
- **Просмотр и экспорт:** Переход в расширенный режим.
    - *Фильтры:* По диапазону дат, S/N.
    - *Настройка колонок:* Выбор, какие поля отображать в таблице.
    - *Экспорт:* Выгрузка отфильтрованных данных в Excel или CSV (можно выгрузить только текущую страницу или все данные).

### 3.2. Результаты опроса (`SensorResults`)
**Назначение:** Мониторинг технического состояния сбора данных.
**Отображает:**
- **Время опроса** датчика.
- **Статус:** Успешно (200) или Ошибка.
- **Время ответа** в миллисекундах.
- **Детали (📊):** Просмотр сырого JSON ответа от датчика. Полезно для отладки.

## 4. Модули работы с данными (Аналитика и Управление)

### 4.1. Графики и диаграммы (`GraphsAndCharts`)
**Назначение:** Визуализация трендов показателей датчиков во времени.
**Как работать:**
1.  **Выберите пост** мониторинга.
2.  **Выберите группу датчиков** (например, DSPD).
3.  **Укажите период:** 24ч, 3д, 7д, 14д, 30д.
4.  **Выберите агрегацию:** "Час" (усреднение по часам), "10 мин" или "Все" (сырые точки).
5.  **Используйте мышь:**
    - *Наведение:* Всплывающая подсказка с точными значениями в конкретный момент времени.
    - *Клик на легенде:* Скрыть/показать конкретную линию на графике.

### 4.2. Карта мониторинга (`MonitoringMap`)
**Назначение:** Геопространственное отображение оборудования.
**Как работать:**
- На карте отображаются иконки постов и датчиков (с учетом GPS-координат, указанных в справочниках).
- **Клик по иконке датчика:** Справа открывается панель с **актуальными показаниями** (последнее полученное значение: температура, ветер, видимость и т.д.).
- **Поиск:** Строка поиска для быстрого нахождения поста на карте.

### 4.3. Отчеты (`Report`)
**Назначение:** Формирование сводных документов.
**Как работать:**
1.  Выберите пост мониторинга.
2.  Укажите период (Дата начала и окончания).
3.  Нажмите "Сформировать".
*(Примечание: На данный момент реализован каркас, ожидается доработка генерации печатных форм).*

### 4.4. Камеры (`Cameras`)
**Назначение:** Подключение и просмотр RTSP видеопотоков.
**Управление:**
- **Список камер:** Карточки с превью последнего снимка.
- **Подключить камеру:** Ввод URL потока (rtsp://...), логина, пароля, привязка к посту.
- **Просмотр (Details):**
    - Прямой видеопоток (изображение обновляется).
    - **PTZ Управление:** Кнопки управления поворотом камеры (если поддерживается).
    - **Снимок:** Кнопка "Сделать снимок" сохраняет текущий кадр в архив.
    - **Архив:** Переход к просмотру всех сохраненных снимков с пагинацией.

## 5. Административные модули
*Доступны только для пользователей с ролью **Admin**.*

### 5.1. Ошибки датчиков (`SensorErrors`)
**Назначение:** Диагностика проблем сбора данных.
**Отображает:** Лог ошибок опроса (Timeout, Bad Request, ошибки парсинга JSON).
**Детали:** При нажатии на запись отображается полный стек ошибки (Trace) и тело запроса/ответа, что помогает разработчику найти причину.

### 5.2. Управление пользователями (`Admin`)
**Назначение:** Контроль доступа к системе.
**Возможности:**
- **Список пользователей:** ID, Логин, Роль (User/Admin), Дата создания.
- **Создать пользователя:** Задать Логин, Пароль и назначить роль.
- **Редактировать:** Изменить роль пользователя.
- **Удалить:** Заблокировать доступ.

### 5.3. Аудит (`Audit`)
**Назначение:** Журнал безопасности.
**Показывает:**
- **Действия:** Кто, когда и что делал (Вход, Создание/Удаление поста, изменение настроек).
- **Изменения данных:** Отслеживание, какое поле в справочнике было изменено (было "Старое значение" -> стало "Новое").
- **Фильтрация:** По типу события, пользователю, сущности, дате.

### 5.4. Настройки (`WorkerConfiguration`)
**Назначение:** Управление параметрами фонового процесса-сборщика (Worker).
**Возможности:**
- Редактирование системных параметров без остановки приложения.
- **Типы полей:** Текст, Число, Чекбокс (true/false).
- **JSON настройки:** Сложные конфигурации редактируются через модальное окно с валидацией JSON-формата.

## 6. Главная страница (Dashboard)
*Путь: Панель навигации -> Главная.*

**Назначение:** "Командирский пульт" для быстрого контроля ситуации.
**Функции:**
- **Список постов** с индикацией активности (зеленый/красный).
- **Развернуть пост:** Клик по заголовку поста.
- **Compact View:** По каждому датчику выводятся ключевые показатели без перехода в детали:
    - *IWS:* Температура, Ветер, Осадки.
    - *DSPD:* Сцепление, Температура дороги, Статус.
    - *MUEKS:* Температура шкафа, Напряжение АКБ.
- **Поиск:** Быстрый поиск поста по названию.
